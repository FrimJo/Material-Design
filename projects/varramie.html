About

This is a project where I worked alone on an Android Application for five weeks during the summer of 2015. The application was developed for the purpose of being used as research foundation for the product owners PHD.

The purpose of this application is for users to use the touchscreen, which creates a circular blob looking liquid, and move the finger around the screen moving the blob. This blob is also shown for all other users of the application, and if two users blobs collide the mobile phone or tablet vibrates, this can also be turned off in the settings menu.

On the server side a Google form can be sent out to all users of the application, containing each users unique identification number, for them to fill out.

Each user of the application can also log in through Facebook and send invites to friends to try out this application.

Network

The communication between the Varramie Application and the Java server, which runs on a PC, is handled through the UDP protocol to which I have written all the packages that is being sent between the application and the server. I choose the UDP protocol in front of the TCP protocol because I wasn't that concerned of packages was being lost and was looking for high performance.

For this part of the project I had a lot of help from a server/client chat program I wrote together with a classmate. From that project I could reuse some classes for the UDP communication.

SQL

For this I wrote a small data access layer for storing data on a SQL server. The server saves all coordinates of movements and collisions together with a identification number for each user to be used in Google forms. This was done to save material to use as basis for the product owners PHD.

LiquidFun

Probably the part of the project I enjoyed the most, but also the trickiest. I used Googles LiquidFun which was originally written to be used with C, but I found a port to Java called JBox2D which I used. At the time the documentation was limited and some functionalities weren't implemented so I found it hard to get it work as I expected, but eventually I found a way to get it behave almost as I wanted.

OpenGL ES

This part was probably the hardest one because of my limited experience with 3D graphic programing. The rendering of the blobs could have been a lot smoother, but since the time was limited I focused on other parts. Though if this project is to be continued I hope to have the time to develop better looking blobs by adding some blur to all the particles inside each blob and then set a threshold value for the alpha to get a smooth blob.
